# 娱乐/文娱行业数据分析标准化工作流 (Workflow)

## 目标与适用场景
当用户输入包含如“帮我看下某部电影表现”、“分析下最近的院线数据”、“做一份XX游戏/剧集报告”等涵盖文娱泛行业诉求时，请自动应用本数据分析执行流。

## 工作流标准化步骤

### 阶段 1：理解与勘探 (Exploration)
1. **明确表结构**：必须使用 `describe_table_tool` 查看目标表（例如 movies 相关的表）字段。通常文娱数据会包含如票房(box_office)、评分(rating)、受众画像(audience)等特征列。
2. **校验数据口径**：重点关注是否有记录“上映日期(release_date)”、“类型(genre)”等，这决定了你能做何种深度的拆解。

### 阶段 2：三维立体分析 (Three-dimensional Analysis)
在出具结论之前，强制使用 SQL 工具跑出这 3 个维度的数据证据：
- **维度A: 内容表现面 (Performance)**：如票房大盘、总体播放量、平均评分。
- **维度B: 趋势特征面 (Trend)**：如有时间序列字段，按周/月汇聚查看其长尾效应。
- **维度C: 内容解构面 (Deconstruction)**：按“题材/导演/品类”分组，寻找差异化特征。如果是单片分析，则对比其与其他同类竞品的数据。

### 阶段 3：可视化呈现强制规范 (Viz SOP)
在回复文娱行业数据时，**必须且至少**包含如下两种图表：
- **基础分布图**：通过 `create_chart_tool` 绘制一幅饼图(pie) 或 水平柱状图(horizontal_bar)，标示其题材/品类分布或票房/播放量核心贡献。
- **演进趋势图**：如果有时间维度，强制附加一幅折线图(line) 展示生命周期走势。

### 阶段 4：深度业务报告建议撰写 (Insights)
1. 用活泼、极度贴合互联网语境的语言概括表现（如：“爆款潜质”、“长尾效应明显”、“核心受众盘稳固”等词汇）。
2. 在输出建议时，必须针对：
   - **宣发策略**：(比如建议在何时、什么渠道加大投放力度)。
   - **内容迭代**：(比如该类题材有哪些微小创新点更受喜爱)。
   
## 生效守则
作为 SQL Agent，只要你明确当前分析针对文娱版块，哪怕用户只是一句“查一下电影表”，你都有责任按上述步骤把图表画全，并输出符合“阶段 4”语言风格的分析点评。
